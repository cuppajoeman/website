Scales

Major

With the major scale you might know the standard way we construct it, which is by taking some root note and then following the following pattern. Once done we call the collections.

W W H W W W H

Where W stands as a Whole Tone and H stands as a Half Tone. So starting on C, we get the following notes:

C D E F G A B C

Another way to write this could be in line with our previous system so we can take our W H pattern and write it as

2 2 1 2 2 2 1

Then instead of taking jumps with respect to the previous note, we could consider them with respect to the root of the scale and get these intervals:

0 2 4 5 7 9 11 0

So if we were to make an F major scale, then we would take 5* (F) and generate the following notes from it

  • 5* + 0 = 5* (F)
  • 5* + 2 = 7* (G)
  • 5* + 4 = 9* (A)
  • 5* + 5 = 10* (Bb)
  • 5* + 7 = 0* (C)
  • 5* + 9 = 2* (D)
  • 5* + 11 = 4* (E)

Due to how we are adding these numbers relative to the root tone which in this case is 5*, then we can call this collection of intervals (0 2 4 5 7 9 11) a relative interval collection or ric for short.

Minor

If we consider the minor scale, we have this pattern

W H W W H W W

In terms of numbers, that is

2 1 2 2 1 2 2

And with respect to the root note, the relative interval collection is

0 2 3 5 7 8 10 0

Relative Major/Minor

Something that's interesting to note is that a minor and major scale can define the same set of notes as each other. The canonical example is that of 0* major and 9* minor, where the 0* major generates the notes 0* 2* 4* 5* 7* 9* 11*, and the 9* minor defines the note collection of 9* 11* 0* 2* 4* 5* 7* which are both the same.

In general, what that's saying is that given some note x*, that x* major is the same as (x-3)* minor, which can be seen in generality as:

Major: (x + 0)*, (x + 2)*, (x + 4)*, (x + 5)*, (x + 7)*, (x + 9)*, (x + 11)*

Minor: (x - 3 + 0)*, (x - 3 + 2)*, (x - 3 + 3)*, (x - 3 + 5)*, (x - 3 + 7)*, (x - 3 + 8)*, (x - 3 + 10)*

Which simplifies down to

(x + 9)*, (x + 11)*, (x + 0)*, (x + 2)*, (x + 4)*, (x + 5)*, (x + 7)*

When re-arranged yields:

(x + 0)*, (x + 2)*, (x + 4)*, (x + 5)*, (x + 7)*, (x + 9)*, (x + 11)*

Thus it is now clear as to why these two relative interval collections which differ can generate a set of notes which do not differ.

Generalizing Relativity

We will end this discussion by taking the result into even more generality, we can further see that the relative interval collection for a minor key is the same relative interval collection as the major, but with 3 added everywhere. Consider:

(Major + 3) = 0 + 3, 2 + 3, 4 + 3, 5 + 3, 7 + 3, 9 + 3, 11 + 3 = 3, 5, 7, 8, 10, 0, 2, = 0 2 3 5 7 8 10 (Minor)

Then it should be arithmetically clear that if we start a minor relative interval collection on the note (x - 3)*, then it then it would construct the same notes as the x* major since every +3 would be cancelled by the -3 when added during the note generation process.

So the most general result here is that if you take a relative interval collection and then add n to everyone of it's intervals, and then generate notes from that modified ric from the note (x - n)*, then that generates the same set of notes as using the original ric generating from x*, as the -n's and the +n's will cancel in the generation process.
